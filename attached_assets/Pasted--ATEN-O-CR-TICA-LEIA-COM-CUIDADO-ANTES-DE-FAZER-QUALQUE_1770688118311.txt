🧩 O QUE O NOVO CATÁLOGO DEVE REPRESENTAR

Cada entrada deve passar a conter no mínimo:

1. resolutionPath (array ordenado)

Lista detalhada do caminho que levou do handler até o endpoint.

Exemplo conceitual:

resolutionPath: [
  { tier: "local", file: "Companies.vue", function: "handleSubmit" },
  { tier: "serviceMap", file: "companies.ts", function: "companiesApi.create" },
  { tier: "inheritance", file: "base-api.ts", function: "BaseApiService.create" },
  { tier: "urlResolution", pattern: "{base}/create" },
  { tier: "architectureMatch", strategy: "WS_OPERATION_BASED" }
]

2. resolutionStrategy

String resumindo quais tiers participaram, por exemplo:

"tier2+tier3+tier6"

3. architectureType

REST_CONTROLLER ou WS_OPERATION_BASED

4. interactionCategory

Um enum:

HTTP

UI_ONLY

STATE_ONLY

Entradas com endpoint null NÃO são erro — são UI_ONLY ou STATE_ONLY.

5. confidence (0–1)

Baseado na estratégia usada (tiers mais profundos = menor confiança relativa)
